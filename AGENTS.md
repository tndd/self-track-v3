# 設計と実装
仕様については`design.md`、具体的な実装計画については`plan.md`を参照すること。

# コミットとブランチ
- マイルストーンごとにブランチを切ること。そしてマイルストーン完了後にはdevブランチへ`--no-ff`でマージすること。
- マージについては、すべてのテストが完了した場合にのみ行うようにすること。
- コミットの単位は意味のある粒度で行うこと。特に一括で大量のファイルをコミットするようなことは厳禁する。
- コミットメッセージは日本語で、実装ついてのタイトルと詳細をきちんと省略せずに書くこと。

# コメントについて
- 関数やクラスなどにはきちんとコメントを日本語で書くこと。コード自体を見なくてもある程度挙動が理解できるように。
- コードに修正を加えたら、コメントの方も修正を忘れないこと。コメントを死文化させてはならない。
