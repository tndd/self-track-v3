# self-track-v3
体調管理をするためのアプリ。
特にやったこととコンディションの因果関係を特定するのが目的。
また日記帳も兼ねてる。

# 要求仕様
## action
服薬やサプリなどの自分で行った行動。
説明変数。

## event
起こったこと。
不眠や頭痛などの出来事で、予測対象。
目的変数とも言える。

## condition
体調を五段階-2~2でつける。デフォルト値は0 (フロントでは1~5にするかも)
その時のコメントやaction, eventを付随させることもできる。
この数値もまた予測対象であり、目的変数。

ただし、この数値は１日に複数回記録される。

## group
action, eventをまとめるためのもの

# Table
## record(体調)
- id
- timestamp
- comment
- value(-2 ~ 2でデフォルト値は0)

## tag
actionもeventもtagでまとめてしまう設計。

[tags (マスタ)]
- id: UUID
- name: VARCHAR (例: "頭痛", "ロキソニン")
- group: STRING('薬', 'サプリ', '運動', '症状' ...) <-- 説明変数か目的変数かは、計算や目的にもよって変わり得る。基本的には'症状'がyになる想定。

[record_tags (中間テーブル)]
- record_id: UUID
- tag_id: UUID
- value: NUMERIC (基本的には1だが、薬などでは2などの数値、あるいは運動の強度で2以上になり得る)

# 技術スタック
基本的にはスマホでの利用を想定。
osはandroidを主眼にしているが、iosにも対応できるようにしたい。
そのためflutterを使用する。

データは端末内に保存し、特定のタイミングでクラウドにバックアップ（オプション）。
そのためFlutterと相性のいいDriftを使用する。
